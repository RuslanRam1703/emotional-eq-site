1\. Назначение документа



Этот документ определяет правила формирования промптов для Claude Code в рамках проекта Emotional EQ.



Цель — обеспечить предсказуемое, контролируемое и итеративное изменение фронтенда.



2\. Принцип работы с Claude Code



Claude Code работает как исполнитель, который:



читает текущий код

применяет изменения

не принимает архитектурных решений

не расширяет задачу самостоятельно

3\. Формат итерации



Каждая итерация должна содержать:



3.1 Контекст

текущий экран или модуль

текущее состояние системы

ограничение из TECH\_REQUIREMENTS

3.2 Цель



Одна конкретная задача.



Примеры:



реализовать UI Presets

добавить hover-анимацию кнопок

связать state с Results экраном

3.3 Область изменений



Обязательно указать:



какие файлы можно менять

какие файлы запрещено трогать

3.4 Ограничения

не менять архитектуру

не добавлять новые зависимости

не создавать новые экраны без разрешения

4\. Принцип атомарности

Факт



Одна итерация = одно изменение поведения системы.



Запрещено:

“сделай весь UI”

“улучши проект”

“добавь анимации везде”

Разрешено:

“реализуй Presets экран”

“добавь state для EQ”

“подключи Results к mock data”

5\. Формат промпта (стандарт)



Каждый промпт должен быть структурирован:



1\. CONTEXT

2\. GOAL

3\. FILES AFFECTED

4\. RULES

5\. EXPECTED OUTPUT

6\. Правила изменения кода



Claude Code должен:



минимально изменять существующий код

сохранять структуру файлов

не дублировать логику

не создавать альтернативные реализации

7\. Приоритеты

При конфликте:

TECH\_REQUIREMENTS.md

UX\_BRIEF.md

UI\_GUIDE.md

PROMPT\_RULES.md

промпт итерации

8\. Контроль качества итерации



Каждое изменение должно соответствовать:



отсутствие регрессий

сохранение UX сценария

сохранение state consistency

отсутствие лишнего кода

9\. Ошибочные паттерны промптов

Плохо:

“сделай красиво”

“улучши интерфейс”

“добавь современный UI”

Правильно:

“реализуй Presets экран согласно UX\_BRIEF”

“подключи mock-films.js к Results”

“добавь state.selectedEmotion”

10\. Definition of Prompt Success



Промпт считается корректным, если:



Claude Code выполняет только одну задачу

результат предсказуем

нет архитектурных изменений

нет побочных эффектов

11\. Итоговая модель управления



Emotional EQ использует:



строго контролируемую AI-итерационную разработку через атомарные промпты
