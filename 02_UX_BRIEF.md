1\. UX-цель системы



UX Emotional EQ направлен на снижение когнитивной нагрузки при выборе контента и замену поиска по каталогу эмоционально-ориентированным сценарием выбора.



Система должна обеспечивать быстрый переход от состояния неопределённости к персонализированной подборке.



2\. Базовый пользовательский сценарий

Movie Card (детализация выбора)

3\. Логика переходов

Onboarding → Presets

происходит автоматически после завершения онбординга

пользователь не принимает решений

Presets → Results

если пользователь выбирает готовый эмоциональный пресет

происходит мгновенный переход к результатам

Presets → Emotional EQ

пользователь выбирает “точную настройку”

система открывает экран эквалайзера

Emotional EQ → Results

пользователь подтверждает настройку

система генерирует подборку

Results → Movie Card

пользователь выбирает фильм из подборки

открывается детальная карточка

4\. Состояния системы

4.1 Loading state

используется при генерации подборки

должен быть минимальным по времени

не блокирует интерфейс дольше необходимого

4.2 Empty state

отсутствуют подходящие фильмы

система предлагает изменить эмоциональные параметры

4.3 Active state

пользователь взаимодействует с пресетами / EQ / карточками

5\. Поведение интерфейса

Принципы:

интерфейс всегда предсказуем

одно действие = один результат

отсутствие скрытых переходов

минимизация когнитивных шагов

6\. Логика эмоционального выбора

Presets

быстрый выбор состояния

минимальная точность

максимальная скорость

Emotional EQ

точная настройка состояния

5 независимых параметров

более высокая когнитивная вовлечённость

7\. Принципы UX

7.1 Линейность



Пользователь всегда движется вперёд, без сложных ветвлений.



7.2 Обратимость



Любое действие может быть изменено до генерации результата.



7.3 Минимизация выбора



В каждый момент времени пользователь принимает только одно решение.



7.4 Контекстность



Каждый следующий экран зависит от предыдущего выбора эмоции.



8\. Ограничения UX

запрещены параллельные сценарии

нельзя вводить дополнительные шаги между основными экранами

нельзя перегружать пользователя настройками

нельзя менять структуру сценария без необходимости

минимальное количество решений → максимальная персонализация результата
