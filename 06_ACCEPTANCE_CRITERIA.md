1\. Назначение документа



Этот документ определяет критерии, по которым оценивается корректность реализации каждой итерации в проекте Emotional EQ.



2\. Общие критерии качества



Каждая итерация считается успешной, если:



функциональность работает согласно UX\_BRIEF

изменения не нарушают существующие экраны

отсутствуют JS ошибки

UI соответствует UI\_GUIDE

state синхронизирован с интерфейсом

3\. UX критерии

Обязательные:

пользователь проходит сценарий без блокеров

переходы между экранами соответствуют UX\_BRIEF

нет лишних шагов или экранов

4\. UI критерии

Обязательные:

соблюдена glassmorphism система

цветовая система не нарушена

компоненты соответствуют UI\_GUIDE

отсутствуют визуальные артефакты

5\. Технические критерии

Обязательные:

код не содержит критических ошибок

state управляет UI

DOM не содержит конфликтующих обновлений

структура файлов сохранена

6\. Критерии итерации



Итерация считается завершённой, если:



выполнена конкретная задача

нет регрессий

не изменена архитектура без необходимости

результат предсказуем

7\. Definition of Done (DoD)



Итерация считается готовой, если:



UI работает во всех сценариях

нет broken states

поведение соответствует UX\_FLOW

код читаем и структурирован

8\. Запрещённые состояния



Итерация считается НЕвалидной, если:



добавлены новые экраны без разрешения

изменена архитектура state без запроса

нарушен UX flow

UI отклоняется от UI\_GUIDE

9\. Итог



Acceptance Criteria — это финальный фильтр качества перед переходом к следующей итерации.
