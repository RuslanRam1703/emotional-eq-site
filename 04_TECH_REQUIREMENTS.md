1\. Техническая модель проекта



Проект Emotional EQ реализуется как:



статический frontend

без backend на текущем этапе

с mock-данными

управляемый через итерации Claude Code

2\. Технологический стек

Обязательный стек:

HTML5 (семантический)

CSS3 (modular architecture)

Vanilla JavaScript (ES6+)

Запрещено на текущем этапе:

React / Vue / Angular

backend-фреймворки

сложные сборщики (если не добавлены явно позже)

внешние UI-библиотеки

3\. Архитектура фронтенда

Структура ответственности:

index.html → entry point

styles/ → визуальная система

scripts/ → логика приложения

assets/ → медиа ресурсы

4\. JavaScript архитектура

4.1 Модули



Приложение должно быть разделено на:



state management

UI rendering

navigation (router)

data layer (mock data)

4.2 State model



Состояние приложения должно включать:



currentScreen (onboarding / presets / eq / results / card)

selectedEmotion

eqValues (5 параметров)

selectedMovie

loadingState

4.3 Принцип управления состоянием

единственный источник истины (single source of truth)

изменения состояния → триггер UI update

запрещено дублирование состояния в DOM

5\. UI rendering

Принцип:



UI строится через:



state → render function → DOM update



Запрещено:

прямые манипуляции DOM без учета state

смешивание логики и UI в одном файле

“магические” inline обработчики

6\. Navigation model



Проект НЕ использует SPA-фреймворк, но:



логика переходов реализуется через router.js

каждый экран = отдельное состояние

Поток:



Onboarding → Presets → EQ → Results → Movie Card



7\. Data layer

mock-films.js содержит:

список фильмов

эмоциональные теги

мета-информацию (жанр, настроение)

8\. CSS архитектура

Разделение:

base.css → reset + глобальные стили

tokens.css → цвета, spacing, radius

components.css → UI компоненты

animations.css → motion system

Принцип:

CSS должен быть модульным

запрещён хаотичный глобальный стиль

все цвета идут через tokens

9\. Анимации

CSS transitions предпочтительнее JS

JS анимации только при необходимости

easing должен быть плавным и ненавязчивым

10\. Производительность

Требования:

минимальный DOM reflow

отсутствие тяжелых библиотек

оптимизированные изображения

ленивые эффекты (lazy interaction logic)

11\. Ограничения Claude Code



Claude Code НЕ должен:



переписывать архитектуру без команды

смешивать UI и state logic

добавлять новые технологии без запроса

изменять структуру файлов произвольно

вводить дополнительные экраны

12\. Правило итераций



Каждая итерация:



затрагивает только одну область

изменяет минимально возможный объем кода

не ломает существующие экраны

13\. Definition of Done (технический)



Итерация считается успешной если:



нет JS ошибок

UI работает на всех экранах

состояние синхронизировано с DOM

адаптивность не нарушена

структура файлов сохранена

14\. Итоговая модель системы



Emotional EQ =



state-driven vanilla frontend с итеративной AI-реализацией интерфейса
