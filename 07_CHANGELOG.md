1\. Назначение документа



Документ фиксирует все изменения проекта Emotional EQ по итерациям.



2\. Формат записи



Каждая итерация фиксируется отдельно:



Iteration XX — \[Название]



Added:

\- ...



Changed:

\- ...



Fixed:

\- ...



Removed:

\- ...

3\. Правила ведения

Обязательно:

каждая итерация должна быть зафиксирована

изменения должны быть атомарными

нельзя объединять несколько итераций

4\. Пример записи



Iteration 01 — Presets Screen



Added:



UI пресетов

базовая сетка карточек



Changed:



state подключен к Presets



Fixed:



исправлена ошибка кликов



Removed:



временные тестовые элементы

5\. Запрещено

описывать “глобальные улучшения”

пропускать итерации

писать неконкретные изменения

6\. Итог



CHANGELOG обеспечивает:



прозрачность изменений

контроль прогресса

возможность отката логики
